# Безопасная авторизация (сервер)
localization:
- [eng](/README.md)  
- [ru](/README.ru.md)  

[клиент](https://github.com/Ruschywez/secure-authorization-client.git)

Это простой проект для само-обучения, созданный, чтобы научиться "как создавать безопасный API".

Когда я создавал проекты в прошлом, они были простые и небезопасные.

Главная проблема этих систем заключалась в чрезмерном доверии клиенту. Сервер никогда не проверял права пользователей и кто-угодно мог получить доступ и информацию от бэк-энда, всего лишь используя простые HTTP запросы и чужой ID.

К примеру, эта проблема встречается в этих моих проектах:

- [hakaton yolka 2026](https://github.com/Ruschywez/hakaton_yolka_2026.git)
- [jkh project](https://github.com/Ruschywez/jkh_project.git)

Помимо этого, я хотел бы попробовать реализовать следующие шутки:

- [ ] верификация аккаунта по электронной почте;
- [ ] логирование;
- [x] шифровать информацию пользователей.

## Как это будет работать?

- [x] создать "сессии" для пользователей;
- [x] предоставлять пользователю уникальный ключ для "сессии";
- [x] получать ключ от пользователя в каждом запросе;
- [x] проверять статус ключа сессии и права пользователя;
- [x] если статус и права правильны, то продолжить работу

## Что тут будет?
Я не хочу создавать слишком сложную систему:

#### Database
- [x] users `пользователи`
- [x] sessions `сессии`
- [x] REALLY SECRET INFORMATION :3 `чувствительная информация пользователя`

#### Server
- [x] CRUD
- [x] Security

#### Будет ли тут что-то ещё?
Также я хотел бы создавать все таблицы базы данных при первом запуске программы.
Я собираюсь использовать SQLite с peewee.

## Стэк

| Технология | Почему это здесь? |
| - | - |
| aiosqlite | SQLite драйвер |
| peewee  | Моя любимая ORM на python |
| types-peewee | Поддержка типов данных из Peewee для PyLance |
| peewee-aio | Поддержка асинхронного кода для peewee |
| fastAPI | Простой HTTP API |
| uvicorn | Сервер |

## Установка

1) клонируйте репозиторий
2) установите зависимости из requirements.txt
3) запустить `create_db.py`

#### Запуск

```cmd
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```
